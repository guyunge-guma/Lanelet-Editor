# Vector Map Builder 功能对比

本文档对比本项目（Lanelet Editor）与 TIER IV 官方 [Vector Map Builder](https://tools.tier4.jp/feature/vector_map_builder.html) 的功能差异。

## 对比概述

Vector Map Builder（VMB）是 TIER IV 提供的免费在线 Lanelet2 地图绘制工具，本项目（Lanelet Editor）是独立部署的私有化版本，两者都用于在点云参考下绘制 Lanelet2 矢量地图，但在功能完整度和使用方式上有显著差异。

## 详细分析

### 本项目优势

1. **数据私有化**：完全本地部署，点云和地图数据不离开内网，适合保密级别高的场景
2. **大点云支持**：基于 Potree LOD 八叉树渲染，可流畅处理 2-3GB 点云
3. **编辑效率工具**：自动吸附、撤销重做、批量操作、碰撞检测等提升标注效率
4. **校验能力更强**：空间间隙检测、几何重叠检测、自相交检测等 VMB 不具备
5. **交互体验优化**：校验问题可点击定位、精细化错误提示、视图模式切换
6. **一键部署**：Docker Compose + 国内源配置，10 分钟完成部署
7. **开源可定制**：Apache-2.0 协议，可根据需求修改

### 本项目劣势

1. **RE 类型覆盖不全**：VMB 支持全部 Lanelet2 RE 类型（停车区、人行道、优先车道等），本项目仅支持 4 种常用类型
2. **RE 不可编辑**：已有 RE 只能删除重建，VMB 可直接修改属性
3. **不支持 MGRS 投影**：仅支持 UTM，VMB 两者都支持
4. **无多用户协同**：与 VMB 一样都是单用户，但 VMB 至少有服务器端会话管理

### 建议改进方向（优先级排序）

| 优先级 | 改进项 | 说明 |
|--------|--------|------|
| P0 | RE 可编辑 | 选中 RE 后可修改属性和关联车道 |
| P0 | 扩展 RE 类型 | 新增停车区、人行道、优先车道等类型 |
| P1 | MGRS 投影支持 | 添加 MGRS 投影选项 |
| P1 | 多用户会话 | 基于 WebSocket 的实时协同（长期） |
| P2 | 车道线自动识别 | 基于 ML 的点云车道线自动提取 |
| P2 | 自动车道连接 | 根据拓扑规则自动建议前驱/后继 |

## 总结

本项目在**部署灵活性、数据隐私、大点云处理、编辑效率工具、校验能力**方面优于 VMB；在 **RE 类型完整度、RE 可编辑性、投影格式**方面有差距。核心地图绘制能力（LineString、Lanelet、拓扑、OSM 导入导出）两者持平。

对于需要在私有环境处理大场景点云、注重编辑效率的场景，本项目是更优选择；对于需要完整 RE 类型覆盖、直接对接 TIER IV 工具链的场景，VMB 更合适。
